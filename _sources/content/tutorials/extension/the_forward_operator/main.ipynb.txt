    "## The `forward` method\n",
    "\n",
    "`forward()` is the function NEML2 calls to evaluate your model. The\n",
    "positional arguments are the inputs in the order you declared them in\n",
    "the schema — each one is already a typed wrapper (`Vec`, `SR2`, …) the\n",
    "framework built for you. Return the outputs in schema order.\n",
    "\n",
    "There's also a `v=None` keyword in the signature — that's the\n",
    "chain-rule hook for first derivatives, covered later on this page.\n",
    "Until you need derivatives, you can leave it alone; calls without\n",
    "`v` get back just the outputs.\n",
    "\n",
    "## Implementation\n",
    "\n",
    "Recall the equation for the projectile model:\n",
    "\n",
    "$$\n",
    "  \\boldsymbol{a} = \\boldsymbol{g} - \\mu \\boldsymbol{v}.\n",
    "$$\n",
    "\n",
    "`self.g` is the gravity vector (a buffer, read directly), and the\n",
    "velocity comes in as the first positional argument. The drag\n",
    "coefficient `mu` is a *parameter*, so it is read through\n",
    "`self._get_param(\"mu\", nl_params, Scalar)` rather than `self.mu` —\n",
    "more on that below. Here is the complete `projectile.py`; the\n",
    "`forward` method is the new part, dissected line by line below:"
   ]

    "%%writefile projectile.py\n",
    "\"\"\"Custom ``ProjectileAcceleration`` model — the running example for the\n",
    "``extension`` tutorial chain. Mirrors the C++ tutorial's projectile in\n",
    "the Python-native model surface.\n",
    "\n",
    "The equation is\n",
    "\n",
    "    a = g - mu * v\n",
    "\n",
    "where ``v`` is the projectile velocity (input), ``a`` is the\n",
    "acceleration (output), ``g`` is the gravitational acceleration vector\n",
    "(buffer; constant), and ``mu`` is the scalar dynamic viscosity\n",
    "(parameter; calibratable).\n",
    "\"\"\"\n",
    "\n",
    "from __future__ import annotations\n",
    "\n",
    "from neml2.factory import register_neml2_object\n",
    "from neml2.models.chain_rule import ChainRuleDict\n",
    "from neml2.models.model import Model\n",
    "from neml2.schema import HitSchema, buffer, input, output, parameter\n",
    "from neml2.types import Scalar, Vec\n",
    "\n",
    "\n",
    "@register_neml2_object(\"ProjectileAcceleration\")\n",
    "class ProjectileAcceleration(Model):\n",
    "    \"\"\"Newton's second law for a projectile in a viscous medium:\n",
    "    ``a = g - mu * v``.\n",
    "    \"\"\"\n",
    "\n",
    "    hit = HitSchema(\n",
    "        input(\"velocity\", Vec, \"Velocity of the projectile\", attr=\"_v_name\"),\n",
    "        output(\"acceleration\", Vec, \"Acceleration of the projectile\"),\n",
    "        buffer(\n",
    "            \"gravitational_acceleration\",\n",
    "            Vec,\n",
    "            \"Gravity vector\",\n",
    "            attr=\"g\",\n",
    "            default=Vec.fill(0.0, -9.81, 0.0),\n",
    "        ),\n",
    "        parameter(\"dynamic_viscosity\", Scalar, \"Dynamic viscosity\", attr=\"mu\"),\n",
    "    )\n",
    "\n",
    "    _v_name: str\n",
    "    g: Vec\n",
    "    mu: Scalar\n",
    "\n",
    "    def forward(  # type: ignore[override]\n",
    "        self,\n",
    "        v_in: Vec,\n",
    "        *nl_params,\n",
    "        v: ChainRuleDict | None = None,\n",
    "    ):\n",
    "        # Read the drag coefficient through ``_get_param`` rather than\n",
    "        # ``self.mu``. ``_get_param`` resolves a static slot from ``self`` or\n",
    "        # a promoted runtime input from ``*nl_params``, so the same forward\n",
    "        # keeps working after ``mu`` is promoted (neml2-compile -p); a bare\n",
    "        # ``self.mu`` would be rejected by the parameter-attribute guard.\n",
    "        mu = self._get_param(\"mu\", nl_params, Scalar)\n",
    "\n",
    "        # Compute the value: a = g - mu * v. ``self.g`` is a buffer (not a\n",
    "        # parameter), so reading it directly is fine.\n",
    "        a = self.g - mu * v_in\n",
    "\n",
    "        # Pure forward: return the typed output and stop.\n",
    "        if v is None:\n",
    "            return a\n",
    "\n",
    "        # First-order chain rule: ∂a / ∂v_in = -mu * I. The closure\n",
    "        # captures the local ``mu`` and receives an incoming tangent V\n",
    "        # (a ``Vec`` shaped like the input), returns the contribution\n",
    "        # to ∂(acceleration)/∂(seed-leaf).\n",
    "        actions = {self._v_name: lambda V: -mu * V}\n",
    "\n",
    "        # ``apply_chain_rule`` returns the v_out dict; pair it with the\n",
    "        # value so the caller can unpack ``(a, v_out)``.\n",
    "        return a, self.apply_chain_rule(v, \"acceleration\", actions, output=a)"
   ]

    "The first line reads the parameter:\n",
    "`mu = self._get_param(\"mu\", nl_params, Scalar)`. Always read a\n",
    "*parameter* this way inside `forward` — never as `self.mu`. A bare\n",
    "`self.mu` is rejected by a runtime guard, because it bypasses\n",
    "`_get_param`'s static-or-promoted dispatch: the moment `mu` is\n",
    "promoted to a runtime input (`neml2-compile -p`) the static\n",
    "`nn.Parameter` no longer exists and the attribute read breaks.\n",
    "`_get_param` works for both static and promoted parameters — it pulls\n",
    "the value from `self` when static and from the `*nl_params` pack when\n",
    "promoted — so the leaf stays promotion-compatible. (Buffers like\n",
    "`self.g` are *not* parameters, so reading them directly is fine.) That\n",
    "is also why the signature is `def forward(self, v_in, *nl_params,\n",
    "v=None)`: the `*nl_params` pack is where promoted parameters arrive.\n",
    "\n",
    "The next line is the physics: `Vec - Scalar * Vec` gives back a `Vec`,\n",
    "batched or not. If `v is None` (the usual case) the method returns and\n",
    "you're done.\n",
    "\n",
    "The `else` branch is the chain-rule hook. `actions` maps each input\n",
    "variable to a small function that takes an incoming tangent\n",
    "(something the same shape as that input) and returns its\n",
    "contribution to the output's tangent. For this model the math is\n",
    "simple: $\\partial \\boldsymbol{a}/\\partial \\boldsymbol{v} = -\\mu I$,\n",
    "so the closure is `lambda V: -mu * V`, capturing the local `mu` we\n",
    "read at the top. `apply_chain_rule` then sums the contribution against\n",
    "any tangents the caller seeded on `v`, without ever building the full\n",
    "Jacobian matrix in memory.\n",
    "\n",

    "## Auto-deriving the chain rule with `request_AD`\n",
    "\n",
    "Hand-writing `actions` is the right call when the local Jacobian is simple, like\n",
    "the projectile's $-\\mu I$. But sometimes it isn't — a constitutive law with many\n",
    "coupled terms, or a **machine-learning surrogate** (a trained `torch.nn.Module`)\n",
    "whose derivative you would never want to differentiate by hand. For those, a leaf\n",
    "can declare `request_AD()` and write *only* the value forward; the framework\n",
    "supplies the first-order chain rule for you by reverse-mode automatic\n",
    "differentiation:\n",
    "\n",
    "```python\n",
    "class MySurrogate(Model):\n",
    "    hit = HitSchema(...)            # same input/output declarations as usual\n",
    "\n",
    "    def __post_init__(self):\n",
    "        self.request_AD()           # auto-derive d(output)/d(input) for all pairs\n",
    "\n",
    "    def forward(self, x, *nl_params):\n",
    "        return self.net(x.data)     # value only -- no `v=` branch, no `actions`\n",
    "```\n",
    "\n",
    "`request_AD()` with no arguments covers every `(output, input)` pair; pass\n",
    "`outputs=[...]` / `inputs=[...]` to auto-derive only a subset and hand-write the\n",
    "rest. The result is indistinguishable from a hand-written chain rule and behaves\n",
    "identically on **every route** — eager (`py-eager` / `cpp-eager`) and\n",
    "AOT-compiled (`py-aoti` / `cpp-aoti` / `cpp-dispatch`). It slots into the *same*\n",
    "forward-mode chain-rule graph the framework already uses: neighbouring analytic\n",
    "leaves keep their hand-written `actions`, and only the request_AD leaf's\n",
    "reverse-mode local Jacobian is traced inline (and lowered through AOTInductor on\n",
    "the compiled routes). The `tests/regression/_fixtures/SurrogateFlowRate.py` fixture\n",
    "is a worked example wrapping a Python surrogate as a NEML2 flow rate.\n",
    "\n",
    "A few things to know:\n",
    "\n",
    "- **First-order only.** `request_AD` supplies the `v=` channel\n",
    "  ($\\partial\\,\\text{out}/\\partial\\,\\text{in}$). A leaf that must provide the\n",
    "  second-order chain rule (i.e. one used inside a `Normality` wrap) still\n",
    "  hand-writes it.\n",
    "- **Reverse-mode under the hood.** It is the one autodiff that survives\n",
    "  `torch.export` → AOTInductor. If your differentiated path uses a saved-output\n",
    "  op (`exp` / `sqrt` / `tanh` / reciprocal), route it through the AOTI-safe\n",
    "  variants in `neml2.types.functions` (e.g. `exp_ad`) so the compiled routes\n",
    "  lower (see the upstream-bug note in that module); eager is unaffected.\n",
    "- **AOTI compile.** Pass `-d` to `neml2-compile` to bake the derivative graph,\n",
    "  exactly as for an analytic model — `request_AD` changes *how* the Jacobian is\n",
    "  computed, not *whether* it is compiled. This holds even for a request_AD leaf\n",
    "  inside an `ImplicitUpdate` residual: the Newton-step / implicit-function-theorem\n",
    "  graphs differentiate the residual (which contains the leaf) and lower the same\n",
    "  way.\n",
