    "## The running example\n",
    "\n",
    "A projectile of velocity $\\boldsymbol{v}$ in a viscous medium feels\n",
    "gravity plus linear drag:\n",
    "\n",
    "$$\n",
    "  \\boldsymbol{a} \\;=\\; \\boldsymbol{g} \\;-\\; \\mu \\, \\boldsymbol{v}.\n",
    "$$\n",
    "\n",
    "So the model needs:\n",
    "\n",
    "- an input — the velocity $\\boldsymbol{v}$ (a `Vec`),\n",
    "- an output — the acceleration $\\boldsymbol{a}$ (also a `Vec`),\n",
    "- a parameter — the drag coefficient $\\mu$ (a `Scalar` we may want to\n",
    "  fit to data later, so it has to be calibratable), and\n",
    "- a buffer — the gravitational acceleration $\\boldsymbol{g}$ (a known\n",
    "  constant of the environment).\n",
    "\n",
    "## The schema\n",
    "\n",
    "A model declares its inputs, outputs, parameters, and buffers in one\n",
    "class-level `hit = HitSchema(...)` block. Each line is a single call\n",
    "to a helper from `neml2.schema`:"
   ]
  },

    "from __future__ import annotations\n",
    "\n",
    "from neml2.factory import register_neml2_object\n",
    "from neml2.models.model import Model\n",
    "from neml2.schema import HitSchema, buffer, input, output, parameter\n",
    "from neml2.types import Scalar, Vec\n",
    "\n",
    "\n",
    "class ProjectileAcceleration(Model):\n",
    "    \"\"\"Projectile acceleration under gravity and linear drag: a = g - mu * v.\"\"\"\n",
    "\n",
    "    hit = HitSchema(\n",
    "        input(\"velocity\", Vec, \"Projectile velocity\"),\n",
    "        output(\"acceleration\", Vec, \"Projectile acceleration\"),\n",
    "        parameter(\"dynamic_viscosity\", Scalar, \"Drag coefficient mu\", attr=\"mu\"),\n",
    "        buffer(\n",
    "            \"gravity\",\n",
    "            Vec,\n",
    "            \"Gravitational acceleration vector\",\n",
    "            attr=\"g\",\n",
    "            default=Vec.fill(0.0, -9.81, 0.0),\n",
    "        ),\n",
    "    )\n",
    "\n",
    "    def forward(self, velocity, *promoted_params, v=None):\n",
    "        raise NotImplementedError"
   ]

    "A few notes on the schema:\n",
    "\n",
    "- The string in each declaration (`\"velocity\"`, `\"dynamic_viscosity\"`,\n",
    "  …) is the name the input file will use.\n",
    "- The third argument is the field's docstring — it shows up in the\n",
    "  auto-generated syntax catalog.\n",
    "- `attr=\"mu\"` on the parameter names its storage slot. **Inside\n",
    "  `forward`** read it through `self._get_param(\"mu\", promoted_params,\n",
    "  Scalar)` — reading `self.mu` directly there is rejected, because it\n",
    "  bypasses parameter promotion (see the note below). Outside `forward`\n",
    "  (for inspection) `self.mu` is fine.\n",
    "- A `buffer` is a constant that travels with the model (e.g., goes to\n",
    "  GPU when the model does) but isn't trainable.\n",
    "\n",
    "## Variable declaration\n",
    "\n",
    "```python\n",
    "input(\"velocity\", Vec, \"Velocity of the projectile\")\n",
    "```\n",
    "\n",
    "`input(name, type_cls, doc)` declares one input variable. `type_cls`\n",
    "is one of the typed wrappers from `neml2.types` — `Scalar`, `Vec`,\n",
    "`R2`, `SR2`, `MRP`, etc. — and tells the framework what shape and\n",
    "algebra to expect. `output(...)` works the same way; it declares an\n",
    "output instead.\n",
    "\n",
    "The name string serves two purposes: it's both the HIT option name\n",
    "the user writes in the input file, AND the default variable name when\n",
    "no rename is supplied. So a fresh instance reports\n",
    "`input_spec = {\"velocity\": Vec}`:"

    "## Parameter declaration\n",
    "\n",
    "```python\n",
    "parameter(\n",
    "    \"dynamic_viscosity\",\n",
    "    Scalar,\n",
    "    \"Dynamic viscosity of the medium\",\n",
    "    attr=\"mu\",\n",
    ")\n",
    "```\n",
    "\n",
    "A parameter declaration takes a name, a type, a docstring, and the\n",
    "attribute it should be exposed under (`mu` here). Other useful options:\n",
    "\n",
    "- `default=...` makes the parameter optional in the input file — if\n",
    "  the user omits it, the schema default is used.\n",
    "- `allow_promotion=True` lets the parameter be promoted to a runtime\n",
    "  input — useful when we want to drive the parameter from another\n",
    "  model's output. See [](tutorials-extension-composition) for the\n",
    "  details.\n",
    "\n",
    "`attr=\"mu\"` names the storage slot — handy when the math uses\n",
    "Greek-letter conventions and we'd rather not spell out\n",
    "`dynamic_viscosity` everywhere. The underlying storage is a\n",
    "`torch.nn.Parameter` (visible in `named_parameters()`).\n",
    "\n",
    ":::{note}\n",
    "**Inside `forward`, read a parameter through\n",
    "`self._get_param(\"mu\", promoted_params, Scalar)` — not `self.mu`.** A bare\n",
    "`self.mu` inside `forward` is rejected by a runtime guard: it bypasses\n",
    "parameter promotion, so it would break the moment the parameter is\n",
    "promoted to a runtime input (`neml2-compile -p`), when the static\n",
    "`nn.Parameter` no longer exists. `_get_param` works for both static\n",
    "and promoted parameters, which is why `forward`'s signature carries the\n",
    "`*promoted_params` pack (`def forward(self, velocity, *promoted_params, v=None)`):\n",
    "promoted parameters arrive there. Buffers are not affected — a buffer\n",
    "like `self.g` is read directly. The next tutorial,\n",
    "[](tutorials-extension-forward), shows this in a real `forward` body.\n",
    ":::\n",
    "\n",
    "Outside `forward` — for inspection, calibration setup, etc. —\n",
    "attribute access via `self.mu` is fine and returns a typed `Scalar`\n",
    "wrapper carrying the same data:"
