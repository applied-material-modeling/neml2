    "(tensor-types-region-views)=\n",
    "# Region views\n",
    "\n",
    "On a raw `torch.Tensor`, `t.unsqueeze(0)` or `torch.sum(t, dim=1)`\n",
    "address axes by absolute position — and once a tensor mixes batch,\n",
    "sub-batch, and base axes, an absolute index is ambiguous and easy to\n",
    "get wrong. NEML2 fixes this by exposing every shape-manipulating op\n",
    "through a **region view**, so each reshape, reduction, or concatenation\n",
    "names the region it acts on."
   ]

    "## The views\n",
    "\n",
    "Four properties select a region; each returns a fresh wrapper, so calls\n",
    "chain:\n",
    "\n",
    "- `t.dynamic_batch` — the dynamic batch region. Ops preserve\n",
    "  `sub_batch_ndim`.\n",
    "- `t.sub_batch` — the sub-batch region. Ops adjust `sub_batch_ndim`.\n",
    "- `t.batch` — the combined region, for read-only `.shape` / `.ndim` and\n",
    "  for `cat`; shape-*changing* ops on it raise (see below).\n",
    "- `t.base` — the base region. Read-only except `transpose` on\n",
    "  square-base types (`R2`, `SSR4`).\n",
    "\n",
    "Every mutable view exposes `.shape`, `.ndim`, `.unsqueeze(dim)`,\n",
    "`.squeeze(dim)`, and `.expand(*shape)`:"
   ]
  },

    "\n",

    "print(\"dynamic_batch.expand(20) ->\", tuple(broadcast.data.shape))\n",
    "\n",
    "tr = R2.identity().base.transpose(-2, -1)\n",
    "print(\"base.transpose(-2, -1)   ->\", tuple(tr.data.shape))"

    "## Why `t.batch` can't be reshaped\n",
    "\n",
    "`t.batch` spans both batch regions at once, so a shape-changing op on it\n",
    "would be ambiguous: should a new axis land in the dynamic region or the\n",
    "sub-batch region? Rather than guess, those ops raise and tell you which\n",
    "view to pick:"

    "## Disambiguating the free functions\n",
    "\n",
    "This is where region views earn their keep. Reductions, concatenation,\n",
    "and interpolation live as **free functions** in `neml2.types`, and they\n",
    "take a *region view* instead of a `dim=` integer. The view tells the\n",
    "function which region the axis index addresses — and lets it update the\n",
    "wrapper's metadata correctly (e.g. dropping `sub_batch_ndim` when a\n",
    "sub-batch axis is reduced away).\n",
    "\n",
    "`sum`, `mean`, and `diff` reduce or difference one region's axis:"
   ]
  },
  {
   "cell_type": "code",
   "execution_count": null,
   "id": "ae174104",
   "metadata": {},
   "outputs": [],
   "source": [
    "from neml2.types import sum, mean, diff, linspace, cat, stack\n",
    "\n",
    "ramp = linspace(Scalar(300.0).sub_batch, Scalar(1200.0).sub_batch, 20)\n",
    "print(\"ramp sub_batch_shape ->\", tuple(ramp.sub_batch_shape))\n",
    "\n",
    "total = sum(ramp.sub_batch, 0)  # reduce the sub-batch axis\n",
    "print(\"sum over sub_batch    ->\", tuple(total.data.shape), \"sub_ndim\", total.sub_batch_ndim)\n",
    "\n",
    "avg = mean(ramp.sub_batch, 0)\n",
    "print(\"mean over sub_batch   ->\", tuple(avg.data.shape))\n",
    "\n",
    "delta = diff(ramp.sub_batch, n=1, dim=-1)\n",
    "print(\"diff along sub_batch  ->\", tuple(delta.sub_batch_shape))"
   ]
