    "## Promoting parameters\n",
    "\n",
    "By default every parameter is baked into the compiled graph as a\n",
    "constant. That's fastest, but it also means we can't change them at\n",
    "runtime. To keep one mutable, promote it at compile time with `-p`.\n",
    "Promoted names are fully qualified: a bare leaf is wrapped in a\n",
    "single-child `ComposedModel` at compile time (matching the eager\n",
    "runtime), so `--model elasticity` exposes its modulus as `elasticity.E`:"

    "Each promoted name adds a small per-call cost (it becomes a graph\n",
    "input rather than a baked constant), but the compiled artifact is\n",
    "typically still much faster than eager. Promotion doesn't work for\n",
    "parameters inside an `ImplicitUpdate` segment — the compiler will\n",
    "refuse them with a clear error.\n",
    "\n",
    "## Trade-offs vs eager\n",
    "\n",
    "|                       | Eager                                  | AOTI                                            |\n",
    "|---                    |---                                     |---                                              |\n",
    "| Per-call overhead     | Python + PyTorch dispatcher per op     | Thin Python shim into pre-compiled C++ kernels  |\n",
    "| Build cost            | Zero                                   | Seconds (Inductor + C++ compile, once)          |\n",
    "| Parameter mutation    | Free — they're live attributes         | Only for `-p` names; rest are baked             |\n",
    "| Autograd              | Yes (full eager AD)                    | No (forward-only; `jvp` / `jacobian` sidecars)  |\n",
    "| Device/dtype          | Switchable at runtime via `.to(...)`   | Pinned at compile time (`--device`, `--dtype`)  |\n",
    "| Portability           | Needs the full Python source           | Needs only `.pt2` + metadata + the C++ runtime  |\n",
    "\n",
    "Rule of thumb: stay in eager mode while we're still iterating on a\n",
    "model. Compile it once we're ready to run it at scale — driver\n",
    "runs, large batch sweeps, the inner loop of a finite-element kernel.\n",
    "\n",
    "## Where to go next\n",
