    "## Loop vs. batched: a numerical experiment\n",
    "\n",
    "How much does the loop actually cost? Evaluate the same elastic model\n",
    "on $N=10{,}000$ strain states two ways — a Python loop over single\n",
    "states and one batched call — and time just the model evaluations:"
   ]
  },
  {
   "cell_type": "code",
   "execution_count": null,
   "id": "db13b7f1",
   "metadata": {},
   "outputs": [],
   "source": [
    "import time\n",
    "\n",
    "N = 10_000\n",
    "batch_data = torch.zeros(N, 6)\n",
    "batch_data[:, 0] = torch.linspace(0.0, 0.01, N)\n",
    "strains = SR2(batch_data)\n",
    "\n",
    "# --- Python loop: one model call per state ---\n",
    "t0 = time.perf_counter()\n",
    "for i in range(N):\n",
    "    model(SR2(batch_data[i]))\n",
    "t_loop = time.perf_counter() - t0\n",
    "\n",
    "# --- Single batched call: one model invocation, N states ---\n",
    "t0 = time.perf_counter()\n",
    "model(strains)\n",
    "t_batch = time.perf_counter() - t0\n",
    "\n",
    "print(f\"python loop: {t_loop * 1e3:8.2f} ms\")\n",
    "print(f\"batched:     {t_batch * 1e3:8.2f} ms\")\n",
    "print(f\"speedup:     {t_loop / t_batch:8.1f}x\")"
   ]
  },
  {
   "cell_type": "markdown",
   "id": "a0ffea77",
   "metadata": {},
   "source": [
    "The cell prints the speedup — large even for a model this trivial\n",
    "on CPU, because the loop pays a constant per-call overhead that the\n",
    "batched call amortizes across the whole batch. On GPU, where each\n",
    "Python-level launch carries extra latency, the gap is usually wider\n",
    "still.\n",
