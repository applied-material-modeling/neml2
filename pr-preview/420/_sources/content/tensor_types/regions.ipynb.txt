    "# Tensor shape regions\n",
    "\n",
    "A raw `torch.Tensor` has one flat shape. A NEML2 wrapper gives that\n",
    "shape *structure*: the trailing axes are the fixed mathematical base,\n",
    "and the leading axes are split into two batch regions that the\n",
    "framework treats differently. Understanding the split is the key to\n",
    "reading any wrapper's shape."

    "## The decomposition\n",
    "\n",
    "A wrapper's `data.shape` reads left-to-right as three contiguous\n",
    "regions:\n",
    "\n",
    "```\n",
    "data.shape == (*dynamic_batch_shape, *sub_batch_shape, *base_shape)\n",
    "              └──── leading ────┘└─ middle (static) ─┘└─ BASE_NDIM ─┘\n",
    "```\n",
    "\n",
    "- **Base shape** — the trailing `BASE_NDIM` axes, fixed by the wrapper\n",
    "  type (`(6,)` for `SR2`, `()` for `Scalar`). These encode the\n",
    "  mathematical structure and never broadcast.\n",
    "- **Sub-batch shape** — a small, *static* batching region for per-site\n",
    "  structure (lookup-table axes, finite-volume cells, slip systems). The\n",
    "  chain-rule machinery treats these axes specially. Default 0 — most\n",
    "  models don't need it.\n",
    "- **Dynamic batch shape** — everything left over. Free-form, sized at\n",
    "  call time, and traced as dynamic by `torch.export` so one compiled\n",
    "  artifact handles every batch size."
   ]
  },

    "## Dynamic vs sub-batch\n",
    "\n",
    "The two batch regions exist precisely because the framework treats them\n",
    "differently:\n",
    "\n",
    "| Region        | Sized at…          | Traced as…   | Broadcasts with…                    | Typical use |\n",
    "| :------------ | :----------------- | :----------- | :---------------------------------- | :---------- |\n",
    "| Dynamic batch | call time          | dynamic dim  | everything                          | every ordinary batch — N material points, time steps |\n",
    "| Sub-batch     | construction time  | static shape | other sub-batches of matching width | per-site structure — interpolation-table axis, FV cell, slip-system axis |\n",
    "\n",
    "Default `sub_batch_ndim = 0`: a model that doesn't need the distinction\n",
    "ignores it, and everything sits in the dynamic region. Sub-batch axes do\n",
    "**not** participate in dynamic-batch broadcasting; they behave like a\n",
    "small extra structural region the chain rule accumulates over. The\n",
    "batching rules are covered in [](tensor-types-batching)."
   ]
  },

    "## Two regions you usually don't construct\n",
    "\n",
    "**An internal K region.** Chain-rule tangents carry an extra leading\n",
    "*K* (seed-direction) region to the left of the dynamic batch. Primal\n",
    "values — everything you build by hand — have `k_ndim == 0`, so the\n",
    "region is absent and you can ignore it; it is managed entirely by the\n",
    "framework's derivative machinery."

    "**A dynamic-base `Tensor`.** Alongside the fixed-base primitives there\n",
    "is a `Tensor` type whose base rank is a *runtime* field, not a class\n",
    "invariant. It is the storage unit for variable-shape Jacobian blocks at\n",
    "the equation-system / solver layer; the same class can stand in for a\n",
    "scalar, an `SR2`-shaped block, or an arbitrary `(rows, cols)` matrix.\n",
    "You rarely construct one directly, but it carries the same region\n",
    "vocabulary with `batch_ndim` / `sub_batch_ndim` as explicit fields:"
