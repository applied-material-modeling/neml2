    "## The pipeline\n",
    "\n",
    "```\n",
    "input.i ──► neml2-compile ──► elasticity_aoti.i             (standalone drop-in stub)\n",
    "                              elasticity/\n",
    "                                cpu/  elasticity_meta.json   (variable layout, dtype, device)\n",
    "                                      elasticity.pt2         (AOTI-compiled kernels)\n",
    "                                      elasticity_jvp.pt2     (per-pair Jacobian graph; only with -d)\n",
    "```\n",
    "\n",
    "`neml2-compile` emits one artifact folder per device (`elasticity/<device>/`)\n",
    "plus a single standalone `elasticity_aoti.i` stub next to it. The stub is a\n",
    "copy of the original input with the `[Models]/elasticity` block replaced by an\n",
    "`AOTIModel` shim that points at the artifact folder via an absolute\n",
    "`artifact_path` — everything else (drivers, settings, tensors) is copied through\n",
    "verbatim, so the stub is a drop-in replacement anywhere a `Driver` consumes a\n",
    "model by name. Compiling for several devices at once (`--device cpu cuda`) adds\n",
    "more `<device>/` subfolders; the loader picks the one matching the device it\n",
    "runs on.\n",
    "\n",

    "## Jacobian and Jacobian-vector product\n",
    "\n",
    "Alongside `forward`, the compiled artifact also exposes `jvp` (outputs\n",
    "plus a directional derivative `J @ v`) and `jacobian` (outputs plus the\n",

    "(`neml2.aoti.Model`). The HIT-loaded shim stores that binding on its\n",
    "`_inner` attribute — this attribute is **not stable public API** and may\n",
    "be renamed in a future release, but it is the only way to reach the\n",
    "binding today:"
   ]

    "# NOTE: _inner is an implementation detail, not stable public API.\n",
    "binding = compiled._inner  # the bare ``neml2.aoti.Model`` runtime\n",
    "\n",
    "# Tangent on the input — same shape as the input itself.\n",
    "strain_t = torch.zeros_like(strain.data)\n",
    "strain_t[:, 0] = 1.0  # probe d(stress)/d(epsilon_xx)\n",
    "\n",
    "out, jvp = binding.jvp({\"strain\": strain.data}, {\"strain\": strain_t})\n",
    "print(\"output[stress] :\", out[\"stress\"][:, :3])\n",

    "## Promoting parameters\n",
    "\n",
    "By default every parameter is baked into the compiled graph as a\n",
    "constant. That's fastest, but it also means we can't change them at\n",
    "runtime. To keep one mutable, promote it at compile time with `-p`.\n",
    "Promoted names are fully qualified: a bare leaf is wrapped in a\n",
    "single-child `ComposedModel` at compile time (matching the eager\n",
    "runtime), so `--model elasticity` exposes its modulus as `elasticity.E`:"

    "Each promoted name adds a small per-call cost (it becomes a graph\n",
    "input rather than a baked constant), but the compiled artifact is\n",
    "typically still much faster than eager. Promotion doesn't work for\n",
    "parameters inside an `ImplicitUpdate` segment — the compiler will\n",
    "refuse them with a clear error.\n",
    "\n",
    "## Trade-offs vs eager\n",
    "\n",
    "|                       | Eager                                  | AOTI                                            |\n",
    "|---                    |---                                     |---                                              |\n",
    "| Per-call overhead     | Python + PyTorch dispatcher per op     | Thin Python shim into pre-compiled C++ kernels  |\n",
    "| Build cost            | Zero                                   | Seconds (Inductor + C++ compile, once)          |\n",
    "| Parameter mutation    | Free — they're live attributes         | Only for `-p` names; rest are baked             |\n",
    "| Autograd              | Yes (full eager AD)                    | No (forward-only; `jvp` / `jacobian` sidecars)  |\n",
    "| Device/dtype          | Switchable at runtime via `.to(...)`   | Pinned at compile time (`--device`, `--dtype`)  |\n",
    "| Portability           | Needs the full Python source           | Needs only `.pt2` + metadata + the C++ runtime  |\n",
    "\n",
    "Rule of thumb: stay in eager mode while we're still iterating on a\n",
    "model. Compile it once we're ready to run it at scale — driver\n",
    "runs, large batch sweeps, the inner loop of a finite-element kernel.\n",
    "\n",
    "## Where to go next\n",
