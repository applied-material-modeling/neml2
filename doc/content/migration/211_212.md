# Migrating from v2.1.1 to v2.1.2 {#migration-211-212}

[TOC]

## Input file changes

### Variable naming: subspace prefixes removed {#variable-naming}

The most pervasive change in v2.1.2 is the removal of `LabeledAxis`. Variables no longer
carry `state/`, `forces/`, `residual/`, or `state/internal/` prefixes anywhere in input files.

Every option that previously accepted a prefixed variable name (e.g. `tensor`, `variable`,
`from`, `to`, `invariant`, `cauchy_stress`, `temperature`, `stress`, etc.) now takes the bare
variable name without any prefix.

### History variables renamed (`~N` suffix) {#history-naming}

"Old state" (history) variables, previously named under `old_state/` or `old_forces/` sub-axes, now
use a `~N` suffix where `N` is the lag order. For example:

| v2.1.1 name    | v2.1.2 name |
| :------------- | :---------- |
| `old_state/S`  | `S~1`       |
| `old_forces/E` | `E~1`       |

If you reference history variables explicitly (e.g. in a `ComposedModel`'s `additional_outputs`
list or a custom model), update those names accordingly.

### `NonlinearSystem`: `unknowns` is now required {#nonlinearsystem-unknowns}

In v2.1.1, `NonlinearSystem` inferred its unknowns from the model's `state` input sub-axis.
In v2.1.2, you must list the unknown variables explicitly:

```python
# v2.1.1
[EquationSystems]
  [eq_sys]
    type = NonlinearSystem
    model = 'implicit_model'
  []
[]
```

```python
# v2.1.2
[EquationSystems]
  [eq_sys]
    type = NonlinearSystem
    model = 'implicit_model'
    unknowns = 'ep Ep'
  []
[]
```

Residual variable names default to `<unknown>_residual` for each unknown. You can override them
with the `residuals` option if needed:

```python
[EquationSystems]
  [eq_sys]
    type = NonlinearSystem
    model = 'implicit_model'
    unknowns = 'phi_P phi_S'
    residuals = 'phi_P_residual phi_S_residual'
  []
[]
```

### `SDTSolidMechanicsDriver` and `LDISolidMechanicsDriver` removed {#sdt-ldi-removed}

These convenience driver types were removed. Use `TransientDriver` directly and supply the
strain/stress controls through the generic `force_SR2_*` options.

Example: strain-control (formerly `SDTSolidMechanicsDriver`):

```python
# v2.1.1
[Drivers]
  [driver]
    type = SDTSolidMechanicsDriver
    model = 'model'
    prescribed_time = 'times'
    prescribed_strains = 'strains'
    save_as = 'result.pt'
  []
[]
```

```python
# v2.1.2
[Drivers]
  [driver]
    type = TransientDriver
    model = 'model'
    prescribed_time = 'times'
    force_SR2_names = 'E'
    force_SR2_values = 'strains'
    save_as = 'result.pt'
  []
[]
```

For mixed-control (formerly `LDISolidMechanicsDriver`), use the same pattern with the
appropriate `force_SR2_names`/`ic_SR2_names` values for your control mode.
